# Payload CMS — დემო ტესტირების სახელმძღვანელო

დოკუმენტი იმისთვის, რომ კლინიკის ადმინისტრატორმა (ან ხელისმომწერმა) გადაამოწმოს, რომ Payload CMS-ის ყველა ფუნქცია სწორად მუშაობს და მართლაც ცვლის საიტს.

**ლოკალური URL-ები (npm run dev):**
- ვებსაიტი: `http://localhost:3000/`
- ადმინ-პანელი: `http://localhost:3000/admin`

**დემო-ავტორიზაცია:**
- Email: `admin@admin.ge`
- პაროლი: `111111`
- 2FA კოდი: ავტენტიფიკატორის აპლიკაცია (Google Authenticator / Authy)

> ⚠️ პროდუქციაში გადასვლამდე აუცილებლად შეცვალე პაროლი და ჩართე ნამდვილი TOTP.
> ლოკალურად TOTP გამორთულია `.env.local`-ში: `TOTP_DISABLED=true`.

## 0. სანამ დაიწყებ — ერთი წუთიანი მომზადება

1. Terminal-ში გაუშვი: `npm run dev`
2. დაელოდე "Ready in X ms" შეტყობინებას
3. გახსენი: `http://localhost:3000` — საიტი უნდა აჩვენოს ქართული მთავარი გვერდი
4. გახსენი: `http://localhost:3000/admin` — შესვლის ფანჯარა უნდა გამოვიდეს

თუ რომელიმე გვერდი არ იხსნება:
- შეამოწმე dev-სერვერი — terminal-ში ჩანდეს მუშაობს თუ არა
- გადახედე `.next/dev/logs/next-development.log` ფაილს
- გაასუფთავე ქეში: წაშალე `.next` დირექტორია და გადატვირთე dev

## 1. ენის გადამრთველის გაგება

Payload-ში ორი სხვადასხვა "ენის" კონცეფციაა — ხშირად ერევა:

### A. ადმინის ინტერფეისის ენა *(ჩვენ ვაკონფიგურეთ მხოლოდ ქართულად)*

- **სად:** ადმინ-პანელის მთელი UI (ღილაკები, sidebar, შეტყობინებები)
- **ენა:** მხოლოდ ქართული (`src/i18n/payload-ka.ts`)
- **გადამრთველი UI-ში:** **არ არსებობს** — და ეს განზრახ ასეა
- **მიზანი:** კლინიკის თანამშრომელი ყოველთვის ქართულ ინტერფეისს ხედავს

### B. შინაარსის ლოკალის გადამრთველი ⚠️ *მთავარი ფუნქცია*

- **სად:** ნებისმიერი ჩანაწერის რედაქტირების გვერდის ზედა ნაწილში — dropdown
- **ვარიანტები:**
  - `ქართული ვერსია (საიტი /ge)`
  - `English version (site /en)`
  - `რუსული ვერსია (საიტი /ru)`
- **რა აკეთებს:** გადამრთველი გადახედს, **რომელი ენის ვერსიას ვარედაქტირებთ ერთსა და იმავე ჩანაწერში**.
- **მაგალითი:** ექიმის სახელი არის ერთი ჩანაწერი, მაგრამ მას სამი ვერსია აქვს — ქართულად, ინგლისურად და რუსულად. ეს dropdown გადახედს, რომელ ვერსიას ცვლი ახლა.

## 3. გლობალები — სრული შემოწმება

ყოველი გლობალისთვის:
1. გახსენი Payload-ში
2. შეცვალე რომელიმე ველი ქართულად, ინგლისურად, რუსულად
3. გადით საიტის შესაბამის გვერდზე და დაადასტურე

> **მნიშვნელოვანი:** თუ CMS ველი ცარიელია, საიტი იყენებს `src/messages/{ge,en,ru}.json`-ში ჩაწერილ ფიქსირებულ ტექსტს (fallback). თუ ცვლი — საიტი ცვლის. თუ ცარიელს დატოვებ — საიტი ფიქსირებულ ტექსტს იძახებს. ეს კარგია, არ ანცვიფრებს რომ "ცარიელია".

## 4. კოლექციები

### 4.1 ექიმები (Doctors)

ლოკაცია: **კონტენტი → ექიმები**

ეტაპები ერთი ექიმის რუსულად სათარგმნად:

1. გახსენი ექიმის ჩანაწერი
2. ლოკალის გადამრთველი → `Русская версия (сайт /ru)`
3. **Identity tab** → `Name` ველში დაწერე რუსული სახელი
4. **Profile tab** → `Specialty`, `Biography`, `Qualifications[]`, `Specializations[]` რუსულად
5. Save
6. გადით `http://localhost:3000/ru/doctors/<slug>` — რუსული ვერსია უნდა გამოჩნდეს

**მნიშვნელოვანი ცვლილებები:**
- Doctra-სინქი **ვერ წერს** რედაქტირებულ ველებზე
- ახალ ექიმს ავტომატურად ვწერთ რუსულ ენაშიც (ინგლისურის კოპია), რომ `/ru` არ აჩვენოს ქართული
- რედაქტორი ცვლის რუსულ ვერსიას ხელით

### 4.2 სერვისები (Services)

ლოკაცია: **კონტენტი → სერვისები**

ლოკალიზირებული ველები: `name`, `description`, `shortDescription`
არალოკალიზებული: `slug`, `category`, `icon`, `image` (იგივე ყველა ენაზე)

ტესტი: გადადი რომელიმე სერვისზე → შეცვალე `name` სამივე ენაში → შემოწმე `/ge/services`, `/en/services`, `/ru/services`

### 4.3 ჩექაფ-პაკეტები (CheckupPackages)

ლოკაცია: **კონტენტი → ჩექაფ-პაკეტები**

ლოკალიზებული: `name`, `description`, `duration`, `includedServices[].service`
არალოკალიზებული: `price`, `currency`, `isFeatured`

### 4.4 შეფასებები (Reviews)

ლოკაცია: **კონტენტი → შეფასებები**

ლოკალიზებული: `author`, `text`
**კურირება:** `published` checkbox — გათიშულია არ გამოჩნდება საიტზე

### 4.5 სიახლეები (News)

ლოკაცია: **კონტენტი → სიახლეები**

ლოკალიზებული: `title`, `excerpt`, `content` (blocks), `author`, SEO ველები
`showOnHomepage` checkbox → მთავარ გვერდზე გამოჩნდება ბოლო 6

## 8. ხშირი პრობლემები

### "ცვლილებას ვერ ვხედავ საიტზე"
- გვერდი დატენე ერთხელ კიდევ (`Ctrl+Shift+R` — hard refresh)
- შეამოწმე ლოკალის გადამრთველი — შეიძლება ქართულ ვერსიას ცვლი მაშინ, როცა `/ru` ხედავ
- დაადასტურე `Save` დააჭირე ცვლილების შემდეგ

### "ცარიელი ველი ვშოვე და დავტოვე ცარიელი — საიტი მაინც აჩვენებს ტექსტს"
- ეს ნორმალურია. ცარიელი CMS = ჩაბრუნდება `src/messages/{ge,en,ru}.json`-ში ჩაწერილ ფიქსირებულ ტექსტს
- რომ შეცვალო — შეავსე ველი და დააჭირე Save

### "Validation Error" Save-ის შემდეგ
- ცარიელია სავალდებულო ველი (`required: true`)
- შეცდომის ტექსტი გვიჩვენებს რომელი ველი — შეავსე და კიდევ ერთხელ Save

### "TOTP კოდი ვერ ვწერ — ვის უნდა შევქმნა"
- პირველი შესვლისას Payload აჩვენებს QR კოდს
- Google Authenticator / Authy აპლიკაცია სკანერებს კოდს
- შემდეგ შესვლისას 6-ციფრიანი კოდი

### "ვერ შემიძლია Logout"
- ზედა მარჯვენა კუთხეში მომხმარებლის ხატულა → Logout
